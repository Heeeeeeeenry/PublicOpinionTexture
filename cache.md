下发备注是允许用户编辑的，所以不是固定值。现在我们来设计后端如何处理。首先，向后端发送数据时，应该携带用户信息。后端接收到数据后，首先从数据库中调取一下信件的详细信息和流转数据，如果信件不处于“待处理”状态，则查看流转记录，向前端返回信息“该信件已被（用户名）处理，现在处于（该信件当前状态）状态。”，然后检查用户单位是否有权下发至指定单位（后端已经写好方法），如果没有，则返回“您无权向该单位下发信件。请联系市局民意智感中心。”，最后对比数据库中的信件详细信息（只对比信件表中除当前信件状态和当前信件处理单位的数据），看看哪那些数据变更了，由什么变成了什么，然后将变更记录下来。
然后创建一个流转字典参数，模板为：

{
"备注": {
    "下发备注":此处填写用户下发备注,
    "变更记录"（如果没有变更，可以不填写）:[
        {
            "变更人":此处填写当前用户姓名,
            "变更字段":"信件分类",
            "变更前值":"咨询求助类 / 案件进度查询 / 行政案件进度查询",
            "变更后值":"咨询政策类 / 案件咨询 / 案件办理进度咨询"
        },
        {
            "变更人":此处填写当前用户姓名,
            "变更字段":"诉求内容",
            "变更前值":数据库中填写的诉求内容,
            "变更后值":用户最终决定的诉求内容
        },
        ...
    ]
},
"操作时间": 当前时间,
"操作类型": "市局下发",
"操作人警号": 用户警号,
"操作前状态": "预处理",
"操作后状态": "已下发",
"操作前单位": 用户单位,
"操作后单位": "分局 / 武强县公安局 / 民意智感中心",
}

创建流转记录字典完成后，加在原来流转记录列表的最后面，最后将所有数据存储至数据库中，向前端返回成功信息。
