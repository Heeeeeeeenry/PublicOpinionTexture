# Checklist

## 前端功能验证
- [ ] 点击"不属实"按钮时，显示确认弹窗
- [ ] 点击"提交"按钮时，显示确认弹窗
- [ ] 确认弹窗样式与下发工作台一致
- [ ] 点击取消按钮，弹窗关闭，不执行操作
- [ ] 点击确定按钮，发送反馈请求

## 前端数据验证
- [ ] 反馈请求包含信件编号
- [ ] 反馈请求包含信件基本信息（群众姓名、手机号、身份证号等）
- [ ] 反馈请求包含信件分类信息
- [ ] 反馈请求包含诉求内容
- [ ] 反馈请求包含专项关注标签
- [ ] 反馈请求包含当前信件状态和处理单位
- [ ] 反馈请求包含录音文件名称列表
- [ ] 反馈请求包含反馈内容
- [ ] 反馈请求包含上传文件名称列表
- [ ] 反馈请求包含诉求情况（"属实"或"不属实"）

## 后端处理验证
- [ ] 后端正确验证信件状态为"正在处理"
- [ ] 信件状态不正确时返回错误信息
- [ ] 后端正确检测信件基本信息变更
- [ ] 变更字段正确更新到数据库
- [ ] 后端正确确定反馈单位
- [ ] 流转记录正确生成
- [ ] 流转记录备注包含诉求情况
- [ ] 流转记录备注包含反馈内容
- [ ] 流转记录备注包含信息变更
- [ ] 文件表正确更新通话录音附件
- [ ] 文件表正确更新办案单位反馈附件
- [ ] 信件状态正确更新为"正在反馈"

## 整体流程验证
- [ ] 提交后信件从处理工作台列表中移除
- [ ] 提交后界面正确刷新
- [ ] 提交后显示成功提示
- [ ] 错误情况下显示错误提示
