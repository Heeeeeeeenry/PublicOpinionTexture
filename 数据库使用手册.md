## 3. 核心表结构详解

### 3.1 信件表

#### 3.1.1 表说明

存储所有民意诉求信件的详细信息，只包含静态信息，不包含状态和流转记录。

#### 3.1.2 字段清单

| 字段名       | 类型           | 必填 | 说明                          |
| ----------- | ------------- | ---- | ---------------------------- |
| 序号         | INT           | 是   | 主键，自增                     |
| 信件编号      | VARCHAR(100)  | 是   | 唯一标识                       |
| 群众姓名      | VARCHAR(50)   | 是   | 来信群众姓名                    |
| 手机号        | VARCHAR(20)   | 否   | 联系电话                       |
| 身份证号      | VARCHAR(18)   | 否   | 身份证号                       |
| 来信时间      | DATETIME      | 是   | 收到来信的时间                   |
| 来信渠道      | VARCHAR(50)   | 是   | 12345/12389/局长信箱等          |
| 信件一级分类   | VARCHAR(50)   | 是   | 一级分类                       |
| 信件二级分类   | VARCHAR(50)   | 是   | 二级分类                       |
| 信件三级分类   | VARCHAR(50)   | 是   | 三级分类                       |
| 诉求内容      | LONGTEXT      | 是   | 群众诉求详细内容                 |
| 专项关注标签   | JSON          | 否   | 专项关注标记(黑恶势力/村霸等)      |

#### 3.1.3 索引信息

| 索引名              | 字段           | 类型     |
| ------------------ | ------------- | -------- |
| PRIMARY            | 序号           | 主键     |
| 信件编号            | 信件编号        | 唯一索引  |
| 索引_信件编号        | 信件编号        | 普通索引  |
| 索引_来信时间        | 来信时间        | 普通索引  |
| 索引_信件一级分类     | 信件一级分类     | 普通索引  |
| 索引_信件二级分类     | 信件二级分类     | 普通索引  |
| 索引_信件三级分类     | 信件三级分类     | 普通索引  |

#### 3.1.4 常用查询示例

```sql
-- 查询指定信件详情
SELECT * FROM 信件表 WHERE 信件编号 = '202403210001';

-- 按一级分类统计
SELECT 信件一级分类, COUNT(*) as 数量 
FROM 信件表 
GROUP BY 信件一级分类;

-- 查询专项关注信件
SELECT * FROM 信件表 
WHERE JSON_CONTAINS(专项关注标签, '"黑恶势力"');

-- 统计今日来信数量
SELECT COUNT(*) FROM 信件表 
WHERE DATE(来信时间) = CURDATE();
```

### 3.2 流转表

#### 3.2.1 表说明

记录信件的当前状态和完整的流转历史。

#### 3.2.2 字段清单

| 字段名    | 类型           | 必填 | 说明                          |
| -------- | ------------- | ---- | ---------------------------- |
| 序号      | INT           | 是   | 主键，自增                     |
| 信件编号   | VARCHAR(100)  | 是   | 关联信件编号                    |
| 信件状态   | VARCHAR(100)  | 是   | 当前信件状态                    |
| 流转记录   | JSON          | 是   | 流转历史列表                    |
| 创建时间   | DATETIME      | 是   | 记录创建时间（默认当前时间）       |
| 更新时间   | DATETIME      | 是   | 记录更新时间（自动更新）          |

#### 3.2.3 索引信息

| 索引名         | 字段      | 类型     |
| ------------- | -------- | -------- |
| PRIMARY       | 序号      | 主键     |
| 索引_信件编号   | 信件编号   | 普通索引  |
| 索引_信件状态   | 信件状态   | 普通索引  |
| 索引_创建时间   | 创建时间   | 普通索引  |

#### 3.2.4 信件状态枚举

| 状态值                | 说明                    |
| -------------------- | ---------------------- |
| 预处理                | 初始状态，等待市局下发      |
| 市局已签收             | 市局已签收信件             |
| 区县局/支队已签收            | 区县局/支队已签收信件            |
| 办理单位已签收          | 办理单位已签收信件          |
| 市局下发至区县局/支队         | 市局已下发至区县局/支队          |
| 市局下发至办理单位       | 市局直接下发至基层（越级下发） |
| 区县局/支队下发至办理单位      | 区县局/支队转发至基层            |   
| 市局正在处理            | 市局直接处理               |
| 区县局/支队正在处理           | 区县局/支队直接处理              |
| 办理单位正在处理         | 基层正在处理               |
| 办理单位已退回至区县局/支队    | 基层退回至区县局/支队            |
| 办理单位已退回至市局      | 基层退回至市局（越级）       |
| 区县局/支队已退回至市局        | 区县局/支队退回至市局            |
| 办理单位已反馈至区县局/支队    | 基层反馈至区县局/支队            |
| 办理单位已反馈至市局      | 基层反馈至市局（越级）       |
| 区县局/支队已反馈至市局        | 区县局/支队反馈至市局            |
| 已办结                 | 处理完毕                  |

#### 3.2.6 常用查询示例

### 3.7 下发权限表

#### 3.7.1 表说明

存储具有信件下发权限的单位信息。只存储**额外情况**，默认规则如下：

- **市局/民意质感中心**：可向任何单位下发（默认允许，无需在表中配置）
- **区县局/支队/民意质感中心**：只能向本单位及下属下发（默认允许，无需在表中配置）
- **其他单位**：无下发权限（默认不允许，如需下发需在表中配置）

#### 3.7.2 字段清单

| 字段名       | 类型           | 必填 | 说明                                  |
| ------------ | ------------- | ---- | ------------------------------------ |
| 序号         | INT           | 是   | 主键，自增                            |
| 单位编码     | VARCHAR(50)   | 是   | 单位唯一编码                          |
| 下发范围     | JSON          | 是   | 可下发的单位编码列表                  |
| 创建时间     | DATETIME      | 是   | 记录创建时间（默认当前时间）          |
| 最后更新时间 | DATETIME      | 是   | 记录最后更新时间（自动更新）          |

#### 3.7.3 索引信息

| 索引名         | 字段      | 类型     |
| ------------- | -------- | -------- |
| PRIMARY       | 序号      | 主键     |
| 单位编码       | 单位编码   | 唯一索引  |

#### 3.7.4 下发范围格式

**向所有单位下发**：
```json
["ALLDEPARTMENT"]
```

**向指定单位下发**（区县局/支队民意质感中心）：
```json
["wqxgaj","wqxgaj-156352","wqxgaj-478802","..."]
```

#### 3.7.5 常用查询示例

```sql
-- 查询具有下发权限的单位
SELECT * FROM 下发权限表;

-- 查询某单位是否有下发权限
SELECT * FROM 下发权限表 WHERE 单位编码 = 'myzgzx';

-- 查询某单位可下发的目标单位列表
SELECT 下发范围 FROM 下发权限表 WHERE 单位编码 = 'WQ_MYZZ_ZX';

-- 插入新的下发权限（市局民意质感中心）
INSERT INTO 下发权限表 (单位编码, 下发范围) 
VALUES ('myzgzx', '["ALLDEPARTMENT"]');

-- 插入新的下发权限（区县局民意质感中心）
INSERT INTO 下发权限表 (单位编码, 下发范围) 
VALUES ('WQ_MYZZ_ZX', '["wqxgaj","wqxgaj-156352","..."]');

-- 更新下发范围
UPDATE 下发权限表 
SET 下发范围 = '["wqxgaj","wqxgaj-156352","wqxgaj-478802"]'
WHERE 单位编码 = 'WQ_MYZZ_ZX';

-- 删除下发权限
DELETE FROM 下发权限表 WHERE 单位编码 = 'zazd';
```

### 3.8 专项关注表

#### 3.8.1 表说明

存储上级或专项行动关注的议题标签，用于对信件进行专项关注标记。这些标签会随时间动态变化，如：

- **涉黑涉恶**：黑恶势力相关案件
- **严重上访风险**：可能造成严重上访情况的案件
- **专项行动标签**：如扫黑除恶专项行动期间的关注标签

**特点**：
- 标签动态变化，可根据上级要求随时增删
- 与信件表通过JSON字段关联，一封信可标记多个专项关注标签
- 用于快速筛选和统计特定类型的信件

#### 3.8.2 字段清单

| 字段名       | 类型           | 必填 | 说明                                  |
| ------------ | ------------- | ---- | ------------------------------------ |
| 序号         | INT           | 是   | 主键，自增                            |
| 专项关注标题 | VARCHAR(200)  | 是   | 专项关注标签名称，如"涉黑涉恶"、"严重上访风险"等 |
| 描述         | TEXT          | 否   | 专项关注标签的详细说明和使用场景      |
| 创建时间     | TIMESTAMP     | 是   | 创建时间（默认当前时间）              |
| 最后更新时间 | TIMESTAMP     | 是   | 最后更新时间（自动更新）              |

#### 3.8.3 索引信息

| 索引名         | 字段      | 类型     |
| ------------- | -------- | -------- |
| PRIMARY       | 序号      | 主键     |

#### 3.8.4 与信件表的关联

信件表的`专项关注标签`字段（JSON类型）存储该信件关联的专项关注标签列表：

```sql
-- 查询标记了"涉黑涉恶"的信件
SELECT * FROM 信件表 
WHERE JSON_CONTAINS(专项关注标签, '"涉黑涉恶"');

-- 查询同时标记了多个专项关注的信件
SELECT * FROM 信件表 
WHERE JSON_CONTAINS(专项关注标签, '"涉黑涉恶"')
  AND JSON_CONTAINS(专项关注标签, '"严重上访风险"');

-- 为信件添加专项关注标签
UPDATE 信件表 
SET 专项关注标签 = JSON_ARRAY_APPEND(专项关注标签, '$', '涉黑涉恶')
WHERE 信件编号 = '202403210001';
```

#### 3.8.5 常用查询示例

```sql
-- 查询所有专项关注标签
SELECT * FROM 专项关注表 ORDER BY 创建时间 DESC;

-- 查询指定专项关注标签
SELECT * FROM 专项关注表 WHERE 序号 = 1;

-- 插入新的专项关注标签
INSERT INTO 专项关注表 (专项关注标题, 描述)
```

## 4. 业务操作指南

### 4.0 下发工作台权限区分

下发工作台根据用户权限级别显示不同的信件：

| 用户权限 | 查看状态 | 数据来源 | 说明 |
| -------- | -------- | -------- | ---- |
| 市局 (CITY) | 预处理 | 流转表 | 查看所有预处理信件，可下发至区县局/支队 |
| 区县局/支队 (DISTRICT) | 市局下发至区县局/支队 | 流转表+下发记录 | 只查看下发到本单位的信件，可继续下发至基层 |

**区县局/支队查询逻辑**：

```sql
-- 1. 查询"市局下发至区县局/支队"状态的信件
SELECT 
    l.*,
    lf.信件状态,
    lf.流转记录
FROM 信件表 l
LEFT JOIN 流转表 lf ON l.信件编号 = lf.信件编号
WHERE lf.信件状态 = '市局下发至区县局/支队'
ORDER BY l.来信时间 DESC;

-- 2. 通过流转记录过滤下发到本单位的信件
-- 在流转记录中查找"下发"操作，检查备注字段是否包含本单位编码
```

### 5.3 按时间统计

```sql
-- 按日统计
SELECT 
    DATE(来信时间) as 日期,
    COUNT(*) as 来信数量
FROM 信件表
WHERE 来信时间 >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY DATE(来信时间)
ORDER BY 日期 DESC;

-- 按月统计
SELECT 
    DATE_FORMAT(来信时间, '%Y-%m') as 月份,
    COUNT(*) as 来信数量
FROM 信件表
WHERE 来信时间 >= DATE_SUB(CURDATE(), INTERVAL 12 MONTH)
GROUP BY DATE_FORMAT(来信时间, '%Y-%m')
ORDER BY 月份 DESC;
```

## 6. 注意事项

### 6.1 索引使用

- 查询时尽量使用索引字段：信件编号、来信时间、信件各级分类
- 避免在索引字段上使用函数，如 `DATE(来信时间)` 会导致索引失效

### 6.2 JSON字段操作

- 查询JSON字段使用 `JSON_CONTAINS()` 函数
- 更新JSON字段使用 `JSON_ARRAY_APPEND()`、`JSON_SET()` 等函数
- 判断JSON数组是否为空使用 `JSON_LENGTH()` 函数

### 6.3 事务处理

- 涉及多表操作时（如更新流转表+文件表），建议使用事务保证数据一致性

### 6.4 表关联关系

```
信件表.信件编号 ←→ 流转表.信件编号 (一对一)
信件表.信件编号 ←→ 文件表.信件编号 (一对一)
```

### 6.5 时区规则

- **所有时间字段均采用上海时区（Asia/Shanghai，UTC+8）**
- 数据库服务器时区设置为上海时区
- 应用程序连接数据库时指定时区为上海时区
- 时间存储格式统一为：`YYYY-MM-DD HH:MM:SS`
- 涉及时间计算时，需确保使用上海时区进行转换

**Django 时区配置示例：**
```python
# settings.py
TIME_ZONE = 'Asia/Shanghai'
USE_TZ = True
```

**MySQL 时区设置：**
```sql
-- 查看当前时区
SELECT @@global.time_zone, @@session.time_zone;

-- 设置为上海时区
SET GLOBAL time_zone = '+8:00';
SET SESSION time_zone = '+8:00';
```
